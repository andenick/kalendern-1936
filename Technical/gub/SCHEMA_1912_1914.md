# 1912 / 1914 Taxeringskalender entry grammar (GUB volumes)

Derived from the GUB OCR text layer (coordinate-split body). Close to Jakob's 1936 spec but
**multi-city** and with OCR-dash noise. Used by `parse_taxkal.py`.

## Entry shape
`Surname, Initials., title(s), Location, municipal_income–state_income`
- **Ditto surname** — a leading dash (`–`/`—`/`-`) repeats the surname above (recursive).
- **Same person, 2nd location** — a leading *double* dash; first name + occupation also dittoed.
- **Wife** — title `hustru` (or `änkefru`/`fru`/`fröken` are their own titles); `hustru` inherits the
  location above.
- **Company** — contains `A.-B.` (aktiebolag) → flag.
- **Income** — two figures; **first = municipal, second = state** taxable income. Either side may be
  absent (leading/trailing dash). May **bump down** to the next physical line.

## Differences from 1936 (must handle)
1. **Multi-city.** The running header names the city/section (Göteborg, Falun, Stockholm/Norrmalm, …).
   Record a best-effort `city` column; carry the last recognised city forward.
2. **Location codes are period-suffixed** (`O.`, `Kh.`, `E.`, `Mt.`) and the code set is per-city, so we
   do **not** validate against the fixed Stockholm parish list — we record the raw location token.
3. **Income dash is OCR-noisy:** `5212 4600` (space), `13515- 15300` / `7591 -8400` (stray hyphen),
   `8/70--16600`. Normalise: take the two number groups in the income field regardless of the separator.
4. **Abbreviations:** `:son`, `:a`, possessive `–:s` continuations, `f. d.` (former), `Fr.`.
5. **Ads** are filtered upstream (font-size + grammar) before parsing.

## Output columns (superset of the 1936 table)
`record_id, volume, year, city, page, region, surname, given_names, title, location,
municipal_income, state_income, is_continuation_surname, is_same_person_2nd_loc, is_wife,
is_company, raw`
